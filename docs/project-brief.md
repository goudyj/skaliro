# Skaliro Project Brief

## Context

Jean wants to build an open source AI learning application where users can
connect with their own AI key or account, then talk directly to an AI coach to
start learning.

The ambition is broader than a single vertical. The same app should eventually
support learning a language, a programming language, school topics, music, or
other skills.

## Core Idea

Skaliro is a conversational learning platform:

1. The user chooses or describes what they want to learn.
2. The AI turns that goal into a path.
3. The user learns through dialogue, exercises, feedback, and repetition.
4. The app tracks progress and adapts the next sessions.
5. Gamification keeps motivation visible without making the app childish.

## Differentiation

Many existing tools already cover parts of this:

- AI tutors such as Khanmigo-style assistants;
- language apps with AI conversation;
- course dashboards and analytics tools;
- open source tutor experiments.

The opportunity is to combine these into a developer-friendly, open source,
bring-your-own-key platform that is not locked to one subject.

## Target Users

- Self-learners who want a guided path instead of random prompting.
- Developers who want to learn languages, frameworks, or concepts by doing.
- Language learners who want conversation plus structure.
- People who need motivation, feedback, and measurable progression.

## Product Principles

- Conversation is the entry point.
- Structure appears when it helps, not before.
- Progress should be visible and useful.
- Gamification should support learning, not replace it.
- The core should stay open and hackable.

## Open Questions

- Should Skaliro start as a web app only, or web plus mobile later?
- Should learning paths be user-generated, AI-generated, or curated first?
- Which AI providers should be supported in the first version?
- How much should voice be part of the MVP?
- Should the app support local models from day one?
